# Notion Invoice Viewer MVP PRD

## 핵심 정보

**목적**: Notion에서 작성한 견적서를 클라이언트가 별도 로그인 없이 웹에서 즉시 확인하고 PDF로 다운로드할 수 있게 한다
**사용자**: 프리랜서 또는 소규모 사업자(어드민)와 해당 견적서를 받는 고객(클라이언트)

---

## 사용자 여정

```
1. [어드민] Notion에서 견적서 작성
   ↓ Notion 데이터베이스에 견적서 저장

2. [어드민] 견적서 목록 페이지 접속 (/invoices)
   ↓ 전체 견적서 목록 확인

3. [어드민] 개별 견적서 상세 페이지에서 공유 링크 복사
   ↓ 클라이언트에게 링크 전달 (이메일, 메신저 등)

4. [클라이언트] 공유 링크로 견적서 상세 페이지 접속 (/invoices/[id])
   ↓ 견적서 내용 확인

   [PDF 다운로드 선택] → PDF 파일 생성 → 로컬 저장
   [그냥 확인만]       → 페이지에서 내용 열람 후 종료

5. [완료] → 클라이언트는 PDF 보관, 어드민은 Notion에서 상태 업데이트
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명                  | 설명                                                                  | MVP 필수 이유                                         | 관련 페이지                            |
| -------- | ----------------------- | --------------------------------------------------------------------- | ----------------------------------------------------- | -------------------------------------- |
| **F001** | Notion 견적서 목록 조회 | Notion API로 견적서 데이터베이스의 모든 견적서를 가져와 목록으로 표시 | 어드민이 전체 견적서 현황을 파악하는 핵심 기능        | 견적서 목록 페이지                     |
| **F002** | 견적서 상세 조회        | 특정 견적서의 항목, 금액, 고객 정보 등 전체 내용을 웹 페이지로 렌더링 | 클라이언트가 견적서를 확인하는 핵심 기능              | 견적서 상세 페이지                     |
| **F003** | PDF 다운로드            | 현재 보고 있는 견적서를 PDF 파일로 변환하여 다운로드                  | 클라이언트가 견적서를 보관/공유하기 위한 필수 기능    | 견적서 상세 페이지                     |
| **F004** | 공유 링크 복사          | 개별 견적서의 고유 URL을 클립보드에 복사                              | 어드민이 클라이언트에게 견적서를 전달하는 유일한 방법 | 견적서 목록 페이지, 견적서 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID       | 기능명               | 설명                                                                   | MVP 필수 이유                                      | 관련 페이지                            |
| -------- | -------------------- | ---------------------------------------------------------------------- | -------------------------------------------------- | -------------------------------------- |
| **F010** | Notion API 연동 설정 | Notion API 키와 데이터베이스 ID를 환경변수로 관리하여 데이터 소스 연결 | 모든 기능의 데이터 소스, 없으면 서비스 불가        | (환경 설정, 페이지 직접 노출 없음)     |
| **F011** | 견적서 데이터 매핑   | Notion 데이터베이스 속성(제목, 금액, 날짜 등)을 웹 UI에 맞게 변환      | 데이터 일관성 보장, 올바른 렌더링을 위한 필수 처리 | 견적서 목록 페이지, 견적서 상세 페이지 |

| **주요 기능**   | • Notion API 호출로 전체 견적서 목록 가져오기 (F001)<br>• 견적서 카드 형태로 목록 렌더링: 제목, 고객명, 금액, 날짜, 상태 표시 (F011)<br>• 각 카드에서 견적서 상세 페이지 링크 제공<br>• 각 카드에 **공유 링크 복사** 버튼 제공 (F004)<br>• 목록 로딩 중 스켈레톤 UI 표시<br>• **더보기** 또는 견적서 카드 클릭 → 견적서 상세 페이지 이동 |

## 데이터 모델

### Invoice (견적서)

| 필드        | 설명                                   | 타입/관계     |
| ----------- | -------------------------------------- | ------------- |
| id          | Notion 페이지 고유 ID                  | string (UUID) |
| title       | 견적서 제목                            | string        |
| clientName  | 고객(클라이언트) 이름 또는 회사명      | string        |
| issueDate   | 견적서 발행일                          | date          |
| dueDate     | 견적서 유효기간 또는 납기일            | date          |
| status      | 견적서 상태 (Draft, Sent, Accepted 등) | string        |
| totalAmount | 총 금액 (세금 포함)                    | number        |
| currency    | 통화 단위 (KRW, USD 등)                | string        |

### InvoiceItem (견적서 항목)

| 필드        | 설명                               | 타입/관계    |
| ----------- | ---------------------------------- | ------------ |
| id          | 항목 고유 식별자                   | string       |
| invoiceId   | 소속 견적서 ID                     | → Invoice.id |
| description | 항목 설명 (서비스명, 작업 내용 등) | string       |
| quantity    | 수량                               | number       |
| unitPrice   | 단가                               | number       |
| amount      | 소계 (quantity × unitPrice)        | number       |

## Notion 데이터베이스 설정 가이드

Notion 데이터베이스는 아래 속성(Property)으로 구성되어야 합니다:

| Notion 속성명 | 속성 타입        | 매핑 필드           |
| ------------- | ---------------- | ------------------- |
| 제목 (Title)  | Title            | invoice.title       |
| 고객명        | Text             | invoice.clientName  |
| 발행일        | Date             | invoice.issueDate   |
| 유효기간      | Date             | invoice.dueDate     |
| 상태          | Select           | invoice.status      |
| 총액          | Number           | invoice.totalAmount |
| 통화          | Select           | invoice.currency    |
| 항목 내역     | Rich Text (JSON) | invoice.items[]     |

> **항목 내역 필드**: 견적서 항목(InvoiceItem) 배열을 JSON 문자열로 저장하거나,
> 별도 Notion 데이터베이스로 분리하여 Relation 속성으로 연결합니다.
